    驱动管理器 的 状态转移
	    状态迁移的原因：
			驱动器内部事件
	        controlword


		T0	start => not ready to switch on
				    bstate: 
				    event:	Reset						
				    astate: 
				    action: drive self-tests and/or self-inirializes.
					
		T1	not ready to switch on => switch on disabled
				    bstate: 
				    event:  drive self-tested and /or initialized   successfully
				    astate: 
				    action: Activate Communication  激活通信(具体是什么内容)
					
		T2	switch on disabled => ready to switch on
				    event:	'Shutdown' command received from host.
				    action: None
		T3	ready to switch on  => switched on
				    event:	'Switch On' command received from host
				    action: The power section is switched on(如果还没 switched on)
		T4	switched on => operation enable
				    event:	'Enable Operation' command received from host
				    action: 使能驱动功能
		T5	operation enable => switched on
				    event:	从 host 收到 'Disable Operation' 命令
				    action: 取消 drive operation
		T6	switched on => ready to switch on
				    event:	从 host 收到 'Shutdown' 命令
				    action: 关闭 power section
		T7	Ready to switch on => switch on disabled
				    event:	从 host 收到 'Quick Stop' and 'Disable Voltage' 命令
				    action: 无
		T8	operation enable => ready to switch on
				    event:	从 host 收到 'Shutdown' 命令
				    action: 立刻关闭 power section, and 如果没法刹住，可以让 电机 空转.
		T9	operation enable => switch on disabled
				    event:	从 host 收到 'Disable Voltage' 命令						
				    action: 同 T8 的 action
		T10	switched on => switch on disabled
				    event: 从 host 收到 'Disable Voltage' 或者 'Quick Stop' 命令							
				    action: 同 T8 的 action
		T11	operation enable => quick stop active
				    event:	从 host 收到 'Quick Stop' 命令
				    action: 执行 quick stop 功能
		T12	quick stop active => switch on disabled
				    event:	'Quick Stop' 执行结束， 或者 从 host 收到 'Disable Voltage' 命令
				    action: 关闭 power section
		T13	all states => fault reaction active
				    event:	驱动 发生了 a fault.
				    action: 执行 appropriate fault reaction
		T14 fault reaction active => fault
				    event:	fault reaction 完成了。
				    action: 取消 drive 功能。 关闭 power section
		T15	fault => switch on disabled
				    event:	从 host 收到 'Fault Reset' 命令
				    action: 如果 驱动中 没有 fault 存在了，则 执行 重置fault condition.
							离开 Fault 状态后，
								 host 主动去 清除 controlword 的 'Fault Reset' 位。
		T16	quick stop active => operation enable
				    event:	从 host 收到 'Enable Operation' 命令
				    action: 使能 驱动功能
    notes:
		如果收到了 引起状态改变 的命令，
		    这个命令 必需 完整处理,
			迁移到新状态后，才能处理下一条命令(next command can be processed)。

	    '驱动功能 被去掉' 意味着，没有能量供给 motor.
		'驱动功能 被使能' 意味着，可以把能量供给 motor.
				  the reference values are processed (torque, velocity, position)
